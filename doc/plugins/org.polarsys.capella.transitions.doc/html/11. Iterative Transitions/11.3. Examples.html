Examples of Iterative Transitions Between Engineering Phases

Basics (Containment and Property Changes)

Overview

Let's consider the following analysis at Operational Analysis:

It can be described in a Operational Activity Interaction Blank:

Performing the First Transition

Let’s perform the first transition (the System Analysis is considered empty, except default model elements)

Perform the Transitions > Functional Transition on the Root System Operational Activity (on the diagram or in the Capella explorer).


You have to create a System Data Flow Blank first.

By drag & drops, it is possible to build the diagram with the same layout at System Analysis

Adding an Element in the Source

Add ‘Assign troops’ Operational Activity in the Operational Analysis

Perform the Transitions > Functional Transition

Perform the Transitions > Functional Transition on the Root System Operational Activity (on the diagram or in the Capella explorer).


Removing an Element in the Source

Remove ‘Assign troops’ Operational Activity in the Operational Analysis

Perform the Transitions > Functional Transition

As the iterative transition is conservative, the target (Logical Architecture) keep the ‘Assign troops’

However there is not incoming or outgoing traceability link on the ‘Assign troops’ System function

Removing an Element in the Target

Remove ‘Assign troops’ function in the System Analysis

Perform the Transitions > Functional Transition on the Root System Operational Activity (on the diagram or in the Capella explorer).

The System Function ‘Assign troops’ is re-created.

Renaming an Element in the Source

Rename ‘Assign troops’ Operational Activity in the Operational Analysis to ‘Assign troops to strategic location’

Perform the Transitions > Functional Transition on the Root System Operational Activity (on the diagram or in the Capella explorer).

The System Function ‘Assign troops’ is' not renamed' (conservative).

Moving an Element in the Source

Move ‘Assign troops’ Operational Activity in the Operational Analysis to ‘Root Operational Activity’

Perform the Transitions > Functional Transition on the Root System Operational Activity (on the diagram or in the Capella explorer).

The System Function ‘Assign troops’ has' not been moved '(conservative).
