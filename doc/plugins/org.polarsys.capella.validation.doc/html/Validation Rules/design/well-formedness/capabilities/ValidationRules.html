Design > Well-formedness > Capabilities


DWF_CA_01 - Capability location
This rule checks that an Capability can only be located in System Context.


DWF_CA_02 - Capability self extension
This rule ensures that a capability cannot extend itself.


DWF_CA_03 - Capability self generalization
This rule ensures that a capability cannot generalize itself.


DWF_CA_04 - Capability self inclusion
This rule ensures that a capability does not include itself.


DWF_CA_06 - Actor involvement in capabilities related to a mission in which it is involved check
This rule checks that the current Actor is involved with the Mission, and then involved in any of its Capabilities.


DWF_CA_07 - Capability check that all the involved Actors are present in at least one Operational Process or Functional Chain or Scenario related to the Capabilities it participates
This rule checks if the involved Actors in the current Capability are present in at least one Operational Process or Functional Chain or Scenario.


DWF_CA_08 - Capability check if the current Capability has Actors that are not involved but are present in at least one Operational Process or Functional Chain or Scenario related to the Capability
This rule checks that if the current Capability has Actors that are not involved but are present in at least one Operational Process or Functional Chain or Scenario related to the Capability.


DWF_CA_09 - Capability check if the current Capability has Functional Chains that are not involved but are related to the Capability
This rule checks that if the current Capability has related Operational Processes or Functionals Chains that are not involved with the Capability.
